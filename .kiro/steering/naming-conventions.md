# Naming Conventions

## Dimension Models

Pattern:
dimension-<entity>.sql

Examples:
- dimension-location.sql
- dimension-customer.sql
- dimension-product.sql

## Fact Models

Pattern:
fact-<entity>.sql

Examples:
- fact-sales.sql
- fact-orders.sql

## SQL Rules

- Use uppercase SQL keywords
- Avoid SELECT *
- Use snake_case column names
- Always format SQL properly
